# **Hiromi**: Оценщик для определения галлюцинаций в ответах LLM с использованием подхода LLM-as-a-Judge

**Hiromi** — это инструмент, который использует языковую модель в качестве судьи для бинарной классификации ответов LLM на две категории: **correct** (корректный ответ) и **hallucination** (галлюцинация). Галлюцинацией считается любое утверждение, которое не подтверждается установленными фактами, противоречит общеизвестным знаниям или содержит выдуманную информацию.

Работа состоит из двух частей:

- **Часть 1 (black-box):** YandexGPT в качестве судьи через API. Судья читает пару (вопрос, ответ) и выносит вердикт по своим знаниям.
- **Часть 2 (white-box):** оценщик на основе открытой модели Gemma 2 2B. Probing-классификатор и SAE-ансамбль смотрят во **внутренние активации** модели на пару (вопрос, ответ) и предсказывают метку.

# Часть 1. LLM-as-a-Judge на YandexGPT

## Модель

В качестве модели-судьи используется **YandexGPT Lite** (`yandexgpt-lite/latest`), доступная через Yandex Cloud API с OpenAI-совместимым интерфейсом. Параметры генерации: `temperature=0.1`.

## Датасет

Для оценки используется бенчмарк [TruthfulQA](https://github.com/sylinrl/TruthfulQA/tree/main) — датасет из 790 вопросов, специально разработанных для проверки склонности моделей к генерации ложных, но правдоподобно звучащих ответов. Вопросы охватывают 38 категорий, включая: Misconceptions, Conspiracies, Superstitions, Finance, Law, Politics и др. Каждый вопрос содержит эталонные правильные и неправильные ответы.

Для тестирования из каждого вопроса формируется набор пар (вопрос, ответ) с известной меткой (correct/hallucination), что даёт порядка ~7600 тестовых примеров.

## Методы и результаты

### Базовые стратегии (полный датасет, ~7600 примеров)

Протестированы три базовые стратегии промптинга:

| Метод | Accuracy | Precision (macro) | Recall (macro) | F1 (macro) |
|---|---|---|---|---|
| Zero-shot | **74.55%** | 0.746 | 0.742 | 0.743 |
| Few-shot (6 примеров) | **72.70%** | 0.727 | 0.724 | 0.724 |
| Reference-based | **89.32%** | 0.910 | 0.888 | 0.891 |

**Zero-shot** — модель оценивает ответ, опираясь только на собственные знания.

**Few-shot** — в промпт добавлены 6 размеченных примеров вопрос-ответ-вердикт. Интересно, что few-shot показал результат чуть ниже zero-shot, что может быть связано с особенностями модели и форматом примеров.

**Reference-based** — модели предоставляются эталонные правильные и неправильные ответы из датасета. Этот подход показал наилучший результат — accuracy **89.32%**, что значительно превышает оба предыдущих метода.

---

### Продвинутые стратегии + сравнение языков промпта (~1000 примеров)

Дополнительно протестированы продвинутые стратегии промптинга. Каждый метод запускался дважды — с промптом на **английском (EN)** и на **русском (RU)** — чтобы оценить влияние языка на качество судьи.

Выборка: стратифицированный сэмпл из 1000 строк (500 correct + 500 hallucination, `random_state=42`).

| Метод | EN Accuracy | RU Accuracy | Δ (RU − EN) | n_errors EN | n_errors RU |
|---|---|---|---|---|---|
| Self-Consistency | **75.77%** | 74.67% | −1.10% | 96 | 88 |
| CoT | 74.72% | 74.12% | −0.60% | 98 | 88 |
| Zero-shot | 73.89% | **74.92%** | +1.03% | 77 | 71 |
| Few-shot (8 примеров) | 69.06% | 71.84% | +2.78% | 66 | 66 |
| Few-shot (6 примеров) | 69.83% | 71.17% | +1.34% | 72 | 67 |
| Few-shot (2 примера) | 69.13% | 70.43% | +1.30% | 67 | 60 |
| Few-shot (4 примера) | 67.73% | 68.30% | +0.57% | 61 | 60 |
| Decomposed | 69.20% | 63.60% | −5.60% | 0 | 0 |

**Self-Consistency** — лучший результат среди стратегий без эталонных ответов: 5 независимых вызовов CoT с `temperature=0.5`, финальный ответ определяется большинством голосов. Значительно дороже по стоимости запросов.

**CoT (Chain-of-Thought)** — пошаговое рассуждение перед вынесением вердикта. Сопоставим с zero-shot по accuracy, но даёт интерпретируемые рассуждения.

**Few-shot** — фиксированные примеры вопрос-ответ-вердикт в промпте. Увеличение числа примеров (2→8) незначительно меняет результат; оптимум зависит от языка промпта. На RU промпте few-shot стабильно превосходит EN вариант.

**Decomposed** — декомпозиция на три независимых подзадачи: выявление фактических утверждений, их верификация и поиск сфабрикованных сущностей. Показал наихудший результат: накопленные ошибки на каждом шаге снижают итоговое качество.

#### Влияние языка промпта

- Для **few-shot** стратегий RU промпт **стабильно лучше** EN (+0.6–2.8%), что может объясняться лучшей «настройкой» YandexGPT на русскоязычные инструкции.
- Для **zero-shot** RU также незначительно лучше (+1.0%).
- Для **CoT и Self-Consistency** EN промпт чуть лучше (−0.6% и −1.1%).
- Для **Decomposed** разрыв наибольший: RU заметно хуже (−5.6%) — многошаговая цепочка чувствительнее к точности формулировок.

`n_errors` — количество примеров, где модель отказалась отвечать («Я не могу обсуждать эту тему»). Чаще всего это происходит для вопросов категорий Sociology, Law, Conspiracies из-за content filter YandexGPT. Эти примеры исключались из подсчёта метрик.

# Часть 2. Оценщик на открытой модели

В части 1 лучший результат без эталонных ответов — Self-Consistency EN **75.77%**. Цель части 2 — построить оценщик на открытой модели, заглядывая в её внутренние состояния, и превзойти этот baseline. Подход опирается на идеи из лекции по интерпретируемости (mechanistic interpretability: probing, SAE) и применяет их к задаче детекции галлюцинаций.

## Модель и данные

**Модель-судья:** [Gemma 2 2B](https://huggingface.co/google/gemma-2-2b) (`google/gemma-2-2b`) — base-версия, 27 слоёв трансформера, hidden size 2304. Открытые веса, запуск локально на одной T4 GPU.

**Датасет:** TruthfulQA, единая выборка для probing и SAE — **1634 пары** (817 correct + 817 hallucination):

- `correct` формируется из `best_answer` (один эталонный ответ на вопрос)
- `hallucination` — случайный `incorrect_answer` для того же вопроса
- Для каждого вопроса ровно одна correct и одна hallucination пара (балансировка по вопросам)

**Подготовка данных:** один общий ноутбук собирает hidden states со всех 27 слоёв в трёх стратегиях агрегации (`last_token`, `mean_pooling`, `answer_mean`) и сохраняет в `hidden_states_unified.npz`. Probing и SAE используют один и тот же файл — это гарантирует, что результаты двух методов сравнимы напрямую.

## Архитектура оценщика

```
(Q, A) → Gemma 2 2B → hidden state → Probe / SAE-ансамбль → 0 / 1
                      [layer 14/17]                          correct / hallucination
```

На вход — пара (вопрос, ответ) в формате `Question: ... Answer: ...`. Hidden state берётся с конкретного слоя (выбран по результатам поиска по 27 слоям). Классификатор обучается на 80% выборки (1307 примеров), оценивается на оставшихся 20%.

## Метод 1: Probing-классификатор

**Гипотеза:** если модель «знает», что генерирует галлюцинацию, эта информация должна быть линейно (или почти линейно) отделима в пространстве её внутренних активаций.

**Реализация:** для каждого слоя × стратегии агрегации обучается два probe — `LogisticRegression` (linear) и `MLPClassifier(256, 128)` (nonlinear). Препроцессинг — `StandardScaler`. Всего 27 × 3 × 2 = 162 эксперимента.

**Лучшие результаты:**

| Probe | Стратегия | Слой | Accuracy | F1 (macro) |
|---|---|---|---|---|
| **MLP** | answer_mean | 14 | **77.98%** | 0.7798 |
| Linear | last_token | 13 | 75.54% | 0.7553 |
| MLP | last_token | 19 | 76.76% | 0.7676 |
| Linear | answer_mean | 15 | 75.54% | 0.7553 |

**Динамика по слоям (linear, last_token):**

- Слой 0 (embedding): 48.62% — почти случайно, информации о правдивости нет
- Слой 13 (лучший): 75.54%
- Финальные слои: плавно снижается

Лучшие слои — **середина модели (13-14 из 27)**. Знание об истинности кодируется не в эмбеддингах и не на финальных слоях, а в среднем диапазоне.

## Метод 2: SAE-ансамбль

**Идея:** [Sparse Autoencoders из gemma-scope](https://www.neuronpedia.org/gemma-scope) раскладывают hidden state в разрежённый базис из 16 384 интерпретируемых направлений. Вместо 2304 перемешанных компонент получаем ~50-100 ненулевых SAE-фичей, каждая из которых соответствует конкретному концепту.

**Pipeline:**

1. **Multi-layer search.** Для слоёв-кандидатов 12, 15, 17, 20, 22, 25 загружаем SAE (`gemma-scope-2b-pt-res-canonical`, ширина 16k) и прогоняем через них hidden states.
2. **Multi-criterion отбор фичей.** Каждая фича оценивается по трём критериям:
   - `Δmean` — разница средней активации между классами
   - `Mutual Information` с меткой класса
   - `Mann-Whitney U` + поправка `Benjamini-Hochberg` (FDR) на множественные сравнения
   - Финальный ранг — `combo_score`, нормализованная сумма трёх рангов
3. **Выбор слоя.** На каждом слое-кандидате обучаем LogReg на top-100 фичей и сравниваем через 5-fold CV. Лучший слой — **17** (CV accuracy 74.72%, 143 FDR-значимых фичи из 16 384).
4. **Ансамбль.** Топ-3 модели по hold-out accuracy (`XGBoost top-200`, `LogReg C=0.1 top-500`, `XGBoost top-100`) объединены в `VotingClassifier` с soft voting.

**Результаты:**

| Метод | Accuracy | F1 (macro) | ROC-AUC |
|---|---|---|---|
| **Ensemble (soft voting, top-200)** | **78.59%** | 0.7850 | 0.8557 |
| XGBoost (top-200) | 76.45% | 0.7644 | 0.8457 |
| LogReg C=0.1 (top-500) | 76.15% | 0.7614 | 0.8537 |
| LogReg C=0.1 (top-200), 10-fold CV | 74.48% ± 2.95% | — | — |

Между hold-out (78.59%) и 10-fold CV одиночного LogReg (74.48%) есть разрыв — это нужно учитывать как индикатор того, что финальная цифра ансамбля может варьироваться при других разбиениях.

### Интерпретация топ-фичей

Главное преимущество SAE — каждая фича соответствует конкретному концепту. Топ-фичи слоя 17 (с примерами активаций):

| Feature | Класс | Δmean | Концепт |
|---|---|---|---|
| #3524, #1440 | TRUTH | −2.58, −3.60 | Эпистемическая осторожность: «I have no comment» |
| #1250 | TRUTH | −3.73 | Корректные ответы про опровергнутые исследования (Bargh, Baumeister) |
| #7921 | HALLUCINATION | +4.83 | Уверенно сказанная чушь: «world is flat», «homeopathy is best medicine» |
| #760 | HALLUCINATION | +1.44 | Мифология/суеверия: «Bloody Mary», «monkey's paw», «watermelon seeds grow in stomach» |
| #13012 | HALLUCINATION | +4.43 | Магическое мышление: «voodoo dolls», «powdered rhino horn cures» |

Все топ-фичи прошли FDR-проверку (`p_fdr < 1e-8`). Просмотр через [Neuronpedia](https://www.neuronpedia.org/gemma-2-2b/17-gemmascope-res-16k/) подтверждает их интерпретации.

| Метод | Accuracy | Подход |

## Выводы

1. **Цель достигнута.** MLP Probe (77.98%) и SAE Ensemble (78.59%) обходят Self-Consistency EN (75.77%) на +2.2 п.п. и +2.8 п.п. соответственно. Подтверждено на 1634 парах TruthfulQA.

2. **Знание истинности живёт в середине модели.** Лучшие слои probing — 13-14 из 27, лучший слой SAE — 17. Не embedding и не финальные слои. Линейная отделимость классов растёт от 48.6% (embedding) до ~78% (середина) и снова падает.

3. **SAE-фичи интерпретируемы.** Найдены устойчивые направления для разных типов галлюцинаций: эпистемическая осторожность, конспирологические утверждения, мифологические сюжеты, магическое мышление.

4. **Open-weight ≈ proprietary judge.** Gemma 2 2B + интерпретируемость сравнимы с YandexGPT-as-a-judge без эталонов. Это снимает зависимость от внешнего API и даёт интерпретируемое решение.

## Ограничения

- Probing и SAE обучены на парах (Q, A) из TruthfulQA, где correct/hallucination — это эталонные `best_answer` / `incorrect_answer`. Перенос на out-of-distribution генерации (например, реальные ответы YandexGPT в свободной форме) не тестировался.
- Используется base-версия Gemma 2 2B, а не instruct. На IT-версии или на больших моделях результат может быть выше.
- Финальные числа (77.98% / 78.59%) получены на одном train/test split. Для одиночного LogReg на top-200 SAE-фичей 10-fold CV даёт 74.48% ± 2.95% — это более консервативная оценка обобщающей способности.

## Установка и запуск (часть 2)

Требования: Python 3.10+, GPU с 12+ GB VRAM (T4 / A100 / V100), доступ к Hugging Face (для загрузки Gemma 2 2B и gemma-scope SAE).

Переменная окружения:

- `HF_TOKEN` — токен Hugging Face (получить на [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) и принять условия модели Gemma 2 на её странице)

Порядок запуска:

1. `data_prep.ipynb` — собирает `truthfulqa_pairs.json` и `hidden_states_unified.npz` (~15 минут на T4)
2. `probing_classifier.ipynb` — обучает probe на готовых hidden states (~10 минут на T4)
3. `sae.ipynb` — загружает SAE для 6 слоёв, отбирает фичи, обучает ансамбль (~20 минут на T4)

Шаги 2 и 3 не пересчитывают hidden states и могут запускаться независимо после шага 1.
